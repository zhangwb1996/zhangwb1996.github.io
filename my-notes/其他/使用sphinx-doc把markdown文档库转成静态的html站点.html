使用 sphinx-doc 把 markdown 文档库转成静态 html 站点

sphinx-doc 可以把 markdown 文件无侵入地转成静态的 html。静态文件可以放到自己的服务器和 cdn 静态站点上；也可以通过 github page 托管到 <github-user>.github.com

配合使用 .rst 文件，可以生成和目录结构相对应的文档导航树。

安装 sphinx-doc

安装 python

https://www.python.org/downloads/

从 PyPI 安装 sphinx-doc

pip install -U sphinx

sphinx-build --version # 检查是否安装成功

安装插件 myst_parser

pip install myst_parser

快速开始

mkdir sphinx-doc  # 创建目录
cd sphinx-doc 	  # 切换到sphinx-doc目录
sphinx-quickstart # 在当前目录创建sphinx doc项目
    > Separate source and build directories (y/n) [n]: y
    > Project name:zhangwb1996.github.io
    > Author name(s):zhangwb1996
    > Project release []:
    > Project language [en]:zh_CN
    # 如果当前目录不为空则会进行如下这步操作
    > Please enter a new file name, or rename the existing file and press Enter [index]:
ls  # build source make.bat Makefile

配置文件

source/conf.py

  • 配置说明参考官方文档

  • html_theme_options

conf.py 文件显示了刚才创建项目时的配置内容，现在如下进行修改。

project = 'zhangwb1996.github.io'
copyright = '2023, zhangwb1996'
author = 'zhangwb1996'

extensions = ['myst_parser'] 	# 启用myst_parser插件

templates_path = ['_templates']
exclude_patterns = [] 		# 生成html时，排除的文件或目录

language = 'zh_CN'

html_theme = 'sphinx_rtd_theme' # html主题
html_static_path = ['_static']

# 添加
html_theme_options = {
    "collapse_navigation": True,
    'collapse_navigation': False,
    'sticky_navigation': True,
    'navigation_depth': 6,
    'includehidden': True,
    'titles_only': False,
}
html_last_updated_fmt = "%b %d, %Y"
html_split_index = True

html_show_sourcelink = False

source_suffix = {
    '.rst': 'restructuredtext',
    '.txt': 'markdown',
    '.md': 'markdown',
}

index.rst

my-notes 及其子目录 下的 index.rst

#新建 index.rst
touch my-notes/index.rst 	# linux
type NUL > my-notes/index.rst 	# windows

  • my-notes：markdown 文档库目录，是 source 的子目录。

  • index.rst：用于生成目录结构，如果需要生成全部目录结构，则每个子目录下都需要以个 index.rst 文件。

  • 子目录的每个 index.rst 配置内容大致分为两种。

    • 嵌套目录：

      c++                     # 页面标题，必填
=====================================
.. toctree::
   :hidden:
   :glob:               # 抓取指令

   c++程序设计语言/index # 子目录-c++程序设计语言的index.rst
   c++11/index          # 子目录-c++11的index.rst
   c++17/index          # 子目录-c++17的index.rst
   *                    # 当前目录下的所有文件

   # 如果c++11目录下全是文件，而且不想生成c++11目录结构
   c++11/*

    • 只有文件：

      c++11
=====================================
.. toctree::
   :hidden:
   :glob:               # 抓取指令

   *                    # 当前目录下的所有文件

source/index.rst

source/index.rst
=====================================

.. toctree::
   :maxdepth: 2
   :hidden:

   /my-notes/index # 此处是相对路径，也可以配置成绝对路径

生成静态 html

·······/sphinx-doc> make clean # 当文档改变对html的影响时全局时，需要先clean

·······/sphinx-doc> make html

生成的静态文件在 source/build/html 目录下

githubpage、github.io

  1. 在 github 新建一个 <username>.github.io 仓库，仓库名需要与 github 的用户名一致

  2. 在 source/build/html 路径下添加 .nojekyll 文件

  3. 使用 git 管理 html 目录，并且关联至远程仓库 <username>.github.io

  4. push

注意事项

make clean 命令会将整个 build 目录（包括 .nojekyll 文件和 .git 目录）清空；为了避免这种情况，可以自己写一个脚本，手动删除。windows 的 bat 脚本 clean.bat 如下：

@REM clean.bat
cd build
rmdir doctrees /Q /S
cd html
@REM rm -r -force *
@REM rmdir -r -force *
del -r -force * /Q
rmdir  my-notes /Q /S
rmdir  _static /Q /S
rmdir  _sources /Q /S
rmdir  _images /Q /S

@REM 生成 .nojekyll 和 .gitignore
type NUL > .nojekyll
type NUL > .gitignore
@REM _sources不是web构建的必须目录，可以在git中忽略
echo _sources > .gitignore

当我们需要重新 make的时候，先执行 ./clean，再执行 make html，最后把 html 仓库推送到 <username>.github.io
