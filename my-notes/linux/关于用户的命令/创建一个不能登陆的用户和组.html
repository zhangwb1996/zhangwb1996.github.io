不能登陆的用户

在 Linux 系统中，可以创建一个不能远程登录的用户，最简单的方法是将该用户的登录 Shell 设置为 /sbin/nologin 或 /bin/false。这样，用户可以在本地登录到系统，但无法通过 SSH 等远程方式登录。以下是创建一个不能远程登录的用户的步骤：

  1. 创建用户：使用 useradd 命令创建用户。例如，创建一个名为 restricteduser 的用户：

    sudo useradd restricteduser

  2. 设置密码（可选）：如果需要，可以为用户设置密码，可以使用 passwd 命令：

    sudo passwd restricteduser

  3. 设置登录 Shell：将用户的登录 Shell 设置为 /sbin/nologin 或 /bin/false。例如：

    sudo usermod --shell /sbin/nologin restricteduser

    或者：

    sudo usermod --shell /bin/false restricteduser

  4. 验证：验证用户是否被正确设置为不能远程登录。你可以尝试使用 SSH 或其他远程登录方式来尝试登录到该用户。如果一切设置正确，你应该无法成功登录到该用户。

通过这些步骤，你就可以在 Linux 系统中创建一个不能远程登录的用户了。

不能登陆的组

要创建一个不能登录的用户组，你可以通过修改 /etc/passwd 文件或者使用 PAM 模块来实现。以下是两种方法的简要说明：

  1. 修改 /etc/passwd 文件：

    你可以将用户的登录 Shell 设置为 /sbin/nologin 或 /bin/false，并将其添加到一个特定的用户组中。这样，属于该用户组的用户将无法登录到系统。

    首先，创建一个新的用户组：

    sudo groupadd nologinusers

    然后，创建一个新用户并将其添加到该用户组，并将其登录 Shell 设置为 /sbin/nologin：

    sudo useradd -g nologinusers -s /sbin/nologin restricteduser

    这将创建一个名为 restricteduser 的用户，并将其添加到 nologinusers 用户组中，并将其登录 Shell 设置为 /sbin/nologin。这样，该用户就无法登录到系统。

  2. 使用 PAM 模块：

    你可以编写一个自定义的 PAM 模块，根据特定的条件来拒绝用户登录。这种方法需要一些额外的工作和深入的了解，并且比较复杂，但可以提供更精细的控制。

    首先，你需要编写一个 PAM 模块，然后将其配置为拒绝登录。这超出了本文的范围，但你可以在网络上找到相关的文档和教程来指导你编写和配置自定义的 PAM 模块。

无论你选择哪种方法，都可以创建一个不能登录的用户组。选择哪种方法取决于你的具体需求和系统配置。

切换到不可登陆用户，执行命令

$ sudo -u restricteduser bash
