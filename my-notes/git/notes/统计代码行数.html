统计代码行数

在 Git 项目中，您可以统计代码行数，同时排除注释行和空白行，以便获得更准确的代码量统计。为此，您可以使用以下步骤：

  1. 使用 Git 命令查看文件变更：首先，使用 Git 命令获取要统计代码行数的文件变更。例如，使用 git ls-files 命令获取所有受 Git 跟踪的文件，或使用 git diff --name-only 获取自上次提交以来发生变更的文件。

  2. 统计代码行数：使用命令行工具，如 wc、grep、awk 等，来统计文件中的代码行数，并排除注释行和空白行。以下是一些命令示例：

    • 使用 grep 排除注释行和空白行，并使用 wc -l 统计行数：

      # 假设您希望统计项目根目录下的所有 .c 文件
git ls-files '*.c' | xargs grep -v '^\s*//' | grep -v '^\s*$' | wc -l

      在这条命令中：

      • git ls-files '*.c'：列出受 Git 跟踪的所有 .c 文件。

      • xargs：将上一条命令的输出作为参数传递给下一条命令。

      • grep -v '^\s*//'：排除以 // 开头的行（C/C++ 注释行）。

      • grep -v '^\s*$'：排除空白行。

      • wc -l：统计行数。

  3. 根据需要修改命令：根据您的项目结构和文件类型，可以修改上述命令。例如，使用不同的文件扩展名、注释样式、或其他命令来适应您的项目。

这些命令可以帮助您统计项目中的代码行数，同时排除注释行和空白行。这有助于您了解项目的实际代码量。如果您希望进一步自动化此过程，可以将这些命令放入脚本中，并根据需要进行调整。

统计代码行数

git ls-files -- '*.cpp' '*.h' | xargs wc -l     # 对应后缀的文件

git ls-files -- | xargs wc -l                   # 全部 track 的文件

git ls-files -- '*.cpp' '*.h' | xargs grep -v '^\s*//' | grep -v '^\s*$' | wc -l

git 命令统计.c .h .lua 有效代码行数，排除 examples，3rd 文件夹

要统计 Git 仓库中.c、.h、.lua文件的有效代码行数，同时排除examples和3rd文件夹，可以使用以下命令：

git ls-files '*.c' '*.h' '*.lua' | grep -vE '^(examples|3rd)/' | xargs wc -l

解释：

  • git ls-files '*.c' '*.h' '*.lua'：列出所有.c、.h和.lua文件。

  • grep -vE '^(examples|3rd)/'：排除examples和3rd文件夹中的文件。

  • xargs wc -l：统计所有剩余文件的行数。

如果你想统计的是有效代码行数（不包括空行和注释），可以使用grep来过滤空行和注释，然后统计行数。这里是一个稍微复杂的例子：

git ls-files '*.c' '*.h' '*.lua' | grep -vE '^(examples|3rd)/' | xargs cat | grep -vE '^\s*$|^\s*//|^\s*#|^\s*\/\*|^\s*\*|^\s*\*/' | wc -l

解释：

  • git ls-files '*.c' '*.h' '*.lua'：列出所有.c、.h和.lua文件。

  • grep -vE '^(examples|3rd)/'：排除examples和3rd文件夹中的文件。

  • xargs cat：将所有剩余文件的内容连接起来。

  • grep -vE '^\s*$|^\s*//|^\s*#|^\s*/\*|^\s*\*|^\s*\*/'：过滤掉空行和常见的注释（包括 C 语言和 Lua 的注释）。

  • wc -l：统计剩余的行数。

请注意，这个命令假设注释只使用单行注释和多行注释的常见模式。根据你的代码风格和注释风格，你可能需要调整过滤注释的正则表达式。
